Документація проєкту генерується автоматично під час запуску процесу збірки (build) або окремої команди генерації документації.

Після запуску відповідного скрипта система сканує вихідний код API, витягує анотації та описані ендпоінти, після чого формує структуровану документацію у форматі OpenAPI/Swagger.

Згенерована документація зберігається у директорії проєкту та може бути переглянута локально через браузер за спеціальною адресою (наприклад /docs), або розгорнута на сервері разом із застосунком.

Таким чином, будь-які зміни в коді API автоматично відображаються в документації після повторної генерації.
